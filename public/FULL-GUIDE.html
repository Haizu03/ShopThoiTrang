4. Kết Nối SQL - Cấu Hình & Cách Hoạt Động

🔌 Cách Kết Nối SQL Hoạt Động

Khi bạn chạy ứng dụng, Node.js cần kết nối tới SQL Server. Quá trình này như sau:

  1. Node.js → ODBC Driver 18 → SQL Server
  2. ODBC Driver 18 là "cầu nối" giữa Node.js và SQL Server
  3. msnodesqlv8 là thư viện Node.js dùng ODBC để kết nối
┌─────────────┐ ┌──────────────────┐ ┌──────────────┐ │ Node.js │ │ ODBC Driver 18 │ │ SQL Server │ │ (app.js) │───────▶│ (TCP/IP Port │───────▶│ (Port 1433) │ │ │ │ 1433 hoặc named │ │ │ │ msnodesqlv8 │ │ pipes) │ │ Database │ └─────────────┘ └──────────────────┘ └──────────────┘

📝 File config/database.js - Chi Tiết

Đây là file cấu hình kết nối SQL. Nó tạo một connection pool (nhóm kết nối tái sử dụng) để tối ưu hiệu năng.

const sql = require('mssql'); const config = { server: '(localdb)\\MSSQLLocalDB', ← Server LocalDB trên máy database: 'shop_thoi_trang', ← Tên database authentication: { type: 'default', options: { userName: 'sa', ← SQL Server Admin (hoặc user khác) password: 'password' ← Mật khẩu (nếu có) } }, options: { encrypt: true, trustServerCertificate: true, ← Tin tưởng chứng chỉ SSL connectionTimeout: 30000, ← Timeout 30 giây requestTimeout: 30000 } } let pool; async function getPool() { if (!pool) { ← Nếu chưa có pool pool = new sql.ConnectionPool(config); ← Tạo mới await pool.connect(); ← Kết nối } return pool; } module.exports = { sql, getPool };
✅ Connection Pool Pattern:
  • Lần đầu: database.js tạo connection pool
  • Lần sau: Tái sử dụng pool cũ (nhanh hơn)
  • Lợi: Không phải tạo connection mỗi lần xử lý request

🔑 Cách Xác Thực SQL Server (Authentication Types)

Cách 1: Windows Authentication (Recommended)

authentication: { type: 'default', options: null ← Dùng Windows login }

✅ Dùng tài khoản Windows của máy

✅ Bảo mật cao (không cần lưu password)

✅ Được khuyến khích cho LocalDB

Cách 2: SQL Server Authentication

authentication: { type: 'default', options: { userName: 'sa', ← SQL Server login password: 'MyPassword' ← SQL Server password } }

❌ Cần mật khẩu (kém bảo mật nếu lưu trong code)

Used when SQL Server admin account (sa) được kích hoạt

🧪 Kiểm Tra Kết Nối SQL Có Hoạt Động

Tạo file test-connection.js để kiểm tra:

const { sql, getPool } = require('./config/database'); async function testConnection() { try { const pool = await getPool(); const result = await pool.request() .query('SELECT 1 AS test'); console.log('✅ Kết nối SQL thành công!'); console.log('Kết quả:', result.recordset); } catch (error) { console.error('❌ Lỗi kết nối:', error.message); } } testConnection();

5. Hướng Dẫn Setup Máy Mới (VS 2022 + LocalDB)

📋 Yêu Cầu Trước Khi Cài

  • Windows 10/11 64-bit
  • Node.js v18+ (https://nodejs.org)
  • Visual Studio 2022 Community (bao gồm LocalDB)
  • ODBC Driver 18 for SQL Server
  • Git (optional, để clone repo)

⚙️ Bước 1: Cài Node.js

  1. Tải Node.js từ https://nodejs.org (LTS version)
  2. Chạy installer, chọn "Add to PATH"
  3. Mở PowerShell/CMD, chạy:
    node --version
    npm --version
  4. Nếu hiển thị version (ví dụ v24.0.0) → OK

⚙️ Bước 2: Cài Visual Studio 2022 + LocalDB

  1. Tải VS 2022 Community từ https://visualstudio.microsoft.com
  2. Chạy installer
  3. Chọn "Data storage and processing" → tick "SQL Server Express LocalDB"
  4. Cài đặt (khoảng 5-10 phút)
  5. Sau cài xong, khởi động lại máy

⚙️ Bước 3: Kiểm Tra LocalDB Có Cài Đúng

  1. Mở SQL Server Management Studio (SSMS) - có trong VS 2022
  2. Hoặc mở PowerShell, chạy:
    sqlcmd -S (localdb)\MSSQLLocalDB -E
  3. Nếu xuất hiện dấu nhắc "1>" → LocalDB cài OK
  4. Gõ exit để thoát

⚙️ Bước 4: Tải Code & Cài Dependencies

  1. Clone repo hoặc tải ZIP:
    git clone https://github.com/phamvy301104-star/Shopthoitrang.git
  2. Vào thư mục dự án:
    cd Shopthoitrang
  3. Cài packages:
    npm install
  4. Chờ khoảng 1-2 phút (tải ~300 packages)

⚙️ Bước 5: Tạo & Import Database

  1. Mở SQL Server Management Studio (SSMS)
  2. Connect tới (localdb)\MSSQLLocalDB (Windows Auth)
  3. Right-click "Databases" → "New Database..."
  4. Gõ tên: shop_thoi_trang → OK
  5. Right-click database vừa tạo → "New Query"
  6. Copy-paste toàn bộ nội dung từ database/schema.sql
  7. Nhấn Ctrl+A → F5 (hoặc Execute) để chạy
  8. Nếu không có error đỏ → Database tạo OK

⚙️ Bước 6: Kiểm Tra config/database.js

Mở file config/database.js, kiểm tra:

const config = { server: '(localdb)\\MSSQLLocalDB', ← Đúng database: 'shop_thoi_trang', ← Đúng (tên database bước 5) authentication: { type: 'default' ← Windows Auth } }

✅ Nếu khác: Sửa cho trùng

⚙️ Bước 7: Khởi Động Server

  1. Mở PowerShell/CMD, vào thư mục dự án:
    cd C:\ShopThoiTrang
  2. Chạy server:
    node app.js
  3. Bạn sẽ thấy:
    Server running on http://localhost:3000
  4. Mở browser, gõ http://localhost:3000
  5. Nếu thấy trang web → ✅ Thành công!

🔐 Bước 8: Đăng Nhập Admin

  1. Trên website, click "Đăng Nhập"
  2. Nhập username & password:
    Username: Lan123 Password: Lan123
  3. Click "Đăng Nhập"
  4. Nếu redirect tới Dashboard → ✅ Login OK!

⚠️ Nếu Gặp Lỗi:

  • "Port 3000 EADDRINUSE" → Port 3000 bị chiếm. Giải pháp:
    taskkill /f /im node.exe
    Rồi chạy lại node app.js
  • "Cannot find module 'express'" → npm install chưa xong. Chạy lại:
    npm install
  • "Login failed for user" → Database config sai. Kiểm tra database.js

6. Chi Tiết Từng File Code

📄 app.js - File Khởi Động Chính

Chức năng: Cấu hình Express, khởi tạo server, setup middleware, kết nối routes

const express = require('express'); const session = require('express-session'); const { getPool } = require('./config/database'); const app = express(); // Middleware cơ bản app.use(express.json()); ← Parse JSON requests app.use(express.urlencoded({ extended: true })); ← Parse form data app.use(express.static('public')); ← Cấp quyền truy cập thư mục public (CSS, JS, images) app.set('view engine', 'ejs'); ← Dùng EJS template engine // Session middleware (lưu trữ thông tin đăng nhập) app.use(session({ secret: 'your-secret-key', resave: false, saveUninitialized: true, cookie: { maxAge: 1000 * 60 * 60 * 24 } ← Session lưu 24 giờ })); // Middleware kiểm tra đăng nhập const isAuthenticated = (req, res, next) => { if (req.session.user) next(); else res.redirect('/dang-nhap'); } const isAdmin = (req, res, next) => { if (req.session.user?.vai_tro === 'admin') next(); else res.status(403).send('Access denied'); } // Import routes (15 files) const authRoutes = require('./routes/auth'); const sanPhamRoutes = require('./routes/sanPham'); // ... thêm 13 routes khác // Đăng ký routes app.use('/api', authRoutes); app.use('/api/san-pham', sanPhamRoutes); // Khởi động server const PORT = process.env.PORT || 3000; app.listen(PORT, () => { console.log(`Server running on http://localhost:${PORT}`); });

📁 Các File Routes Chi Tiết

1. routes/auth.js - Authentication

Chức năng: Xử lý Đăng Nhập, Đăng Ký, Đăng Xuất, Đổi Mật Khẩu

const express = require('express'); const bcrypt = require('bcryptjs'); const { sql, getPool } = require('../config/database'); const router = express.Router(); // POST /dang-nhap (Login) router.post('/dang-nhap', async (req, res) => { try { const { tenDangNhap, matKhau } = req.body; const pool = await getPool(); // Truy vấn tìm user const result = await pool.request() .input('tenDangNhap', sql.VarChar(50), tenDangNhap) .query(` SELECT * FROM tai_khoan WHERE ten_dang_nhap = @tenDangNhap `); if (result.recordset.length === 0) { return res.json({ success: false, message: 'Username not exist' }); } const user = result.recordset[0]; // So sánh mật khẩu với bcrypt const isValidPassword = await bcrypt.compare(matKhau, user.mat_khau); if (!isValidPassword) { return res.json({ success: false, message: 'Wrong password' }); } // Lưu user vào session req.session.user = { tai_khoan_id: user.tai_khoan_id, ten_dang_nhap: user.ten_dang_nhap, vai_tro: user.vai_tro }; res.json({ success: true, message: 'Login successful' }); } catch (error) { res.status(500).json({ error: error.message }); } }); // POST /dang-ky (Register) router.post('/dang-ky', async (req, res) => { try { const { tenDangNhap, matKhau, email } = req.body; const pool = await getPool(); // Mã hóa mật khẩu (salt = 10) const hashedPassword = await bcrypt.hash(matKhau, 10); // Insert user mới (role = 'nguoidung') await pool.request() .input('tenDangNhap', sql.VarChar(50), tenDangNhap) .input('matKhau', sql.VarChar(255), hashedPassword) .input('email', sql.VarChar(100), email) .input('vaiTro', sql.VarChar(20), 'nguoidung') .query(` INSERT INTO tai_khoan (ten_dang_nhap, mat_khau, email, vai_tro) VALUES (@tenDangNhap, @matKhau, @email, @vaiTro) `); res.json({ success: true, message: 'Register successful' }); } catch (error) { if (error.message.includes('UNIQUE')) { res.status(400).json({ error: 'Username or email already exists' }); } else { res.status(500).json({ error: error.message }); } } }); module.exports = router;

2. routes/sanPham.js - Quản Lý Sản Phẩm

Chức năng: GET danh sách, POST thêm, EDIT, DELETE sản phẩm

const express = require('express'); const multer = require('multer'); const { sql, getPool } = require('../config/database'); const router = express.Router(); // Cấu hình Multer (upload ảnh) const upload = multer({ dest: 'public/uploads/products/', limits: { fileSize: 5 * 1024 * 1024 } ← Tối đa 5MB }); // GET /san-pham (Lấy toàn bộ sản phẩm) router.get('/', async (req, res) => { try { const pool = await getPool(); const result = await pool.request() .query(` SELECT sp.*, dm.ten_danh_muc FROM san_pham sp LEFT JOIN danh_muc dm ON sp.danh_muc_id = dm.danh_muc_id ORDER BY sp.ngay_tao DESC `); res.json(result.recordset); } catch (error) { res.status(500).json({ error: error.message }); } }); // POST /san-pham (Thêm sản phẩm mới) router.post('/', upload.single('image'), async (req, res) => { try { const { tenSanPham, danhMucId, gia, soLuong, moTa, gioiTinh } = req.body; const imagePath = req.file ? req.file.filename : null; const pool = await getPool(); await pool.request() .input('tenSanPham', sql.NVarChar(100), tenSanPham) .input('danhMucId', sql.Int, danhMucId) .input('gia', sql.Decimal(10,2), gia) .input('soLuong', sql.Int, soLuong) .input('moTa', sql.NVarChar(500), moTa) .input('hinh_anh', sql.VarChar(255), imagePath) .input('gioiTinh', sql.VarChar(10), gioiTinh) .query(` INSERT INTO san_pham (ten_san_pham, danh_muc_id, gia, so_luong_ton, mo_ta, hinh_anh, gioi_tinh) VALUES (@tenSanPham, @danhMucId, @gia, @soLuong, @moTa, @hinh_anh, @gioiTinh) `); res.json({ success: true, message: 'Product added' }); } catch (error) { res.status(500).json({ error: error.message }); } }); module.exports = router;

7. Giải Thích database.js Chi Tiết

Cấu Trúc Connection Pool

const sql = require('mssql'); // mssql = package Node.js để kết nối SQL Server const config = { server: '(localdb)\\MSSQLLocalDB', // Tên máy chủ SQL (LocalDB trên máy local) // Nếu MSSQL: 'LAPTOP-ABC', 'localhost', '127.0.0.1' // Nếu Azure: 'server.database.windows.net' database: 'shop_thoi_trang', // Tên database cần kết nối authentication: { type: 'default', // 'default' = Windows Authentication // options: null (dùng tài khoản Windows hiện tại) }, options: { encrypt: true, // Mã hóa kết nối (TLS/SSL) trustServerCertificate: true, // Tin tưởng chứng chỉ SSL của server (quan trọng cho LocalDB) connectionTimeout: 30000, // Chờ tối đa 30 giây để kết nối requestTimeout: 30000 // Chờ tối đa 30 giây cho mỗi query } }; let pool = null; // Biến global lưu connection pool (Singleton pattern) async function getPool() { if (!pool) { // Nếu chưa có pool pool = new sql.ConnectionPool(config); // Tạo pool mới với config await pool.connect(); // Kết nối tới SQL Server (async - chờ) console.log('Connected to database!'); } return pool; // Trả về pool để routes dùng } module.exports = { sql, getPool }; // Export để routes import: const { sql, getPool } = require('./config/database');

Cách Dùng database.js Trong Routes

// File: routes/sanPham.js const { sql, getPool } = require('../config/database'); // Import getPool function router.get('/san-pham', async (req, res) => { try { const pool = await getPool(); // Lấy connection pool (nếu chưa có thì tạo, nếu có thì tái sử dụng) const result = await pool.request() .input('danhMucId', sql.Int, 1) .query(`SELECT * FROM san_pham WHERE danh_muc_id = @danhMucId`); // Thực thi query với parameterized input (bảo vệ SQL injection) console.log(result.recordset); // recordset = mảng dòng dữ liệu trả về res.json(result.recordset); } catch (error) { console.error('Database error:', error.message); res.status(500).json({ error: error.message }); } });

🔒 Bảo Vệ SQL Injection

❌ Cách SAI (Dễ bị hack):

// KHÔNG BAO GIỜ DÙNG CÁCH NÀY const userId = req.body.id; const query = `SELECT * FROM tai_khoan WHERE tai_khoan_id = ${userId}`; // Nếu userId = "1; DROP TABLE tai_khoan; --" // → Query trở thành: SELECT * FROM tai_khoan WHERE tai_khoan_id = 1; DROP TABLE tai_khoan; -- // → Database bị xóa!

✅ Cách ĐÚNG (Parameterized Queries):

const userId = req.body.id; const pool = await getPool(); const result = await pool.request() .input('userId', sql.Int, userId) ← Input tách biệt .query('SELECT * FROM tai_khoan WHERE tai_khoan_id = @userId'); ← @userId placeholder // userId được SQL Server coi là DATA, không phải CODE // Nên "1; DROP TABLE" chỉ là string, không execute

🚀 Ví Dụ: Flow Đặt Hàng Hoàn Chỉnh

Khi user nhấn "Đặt Hàng":

1. Frontend (form đặt hàng) gửi POST request tới /dat-hang ↓ 2. Middleware kiểm tra isAuthenticated (req.session.user có không) ↓ 3. Routes/user.js xử lý POST /dat-hang: ├─ Validate input (địa chỉ, giỏ hàng trống không) ├─ Kiểm tra mã giảm giá (nếu có) ├─ Tính tổng tiền ├─ Bắt đầu Transaction (BEGIN TRANSACTION) │ ├─ INSERT vào don_hang │ ├─ INSERT chi_tiet_don_hang (từng dòng giỏ hàng) │ ├─ UPDATE san_pham.so_luong_ton (trừ số lượng) │ ├─ UPDATE ma_giam_gia.da_su_dung (nếu có mã) │ └─ COMMIT (nếu thành công) hoặc ROLLBACK (nếu lỗi) ├─ Xóa session giỏ hàng └─ Trả về JSON: { success: true, don_hang_id: ... } ↓ 4. Frontend nhận response, chuyển hướng tới /don-hang-cua-toi ↓ 5. Routes/user.js GET /don-hang-cua-toi lấy đơn hàng từ DB ├─ Query JOIN: don_hang + chi_tiet_don_hang + san_pham └─ Render view với dữ liệu

10. Các Truy Vấn SQL Quan Trọng

Lấy Sản Phẩm Với Danh Mục

SELECT sp.san_pham_id, sp.ten_san_pham, sp.gia, sp.so_luong_ton, dm.ten_danh_muc, sp.hinh_anh FROM san_pham sp LEFT JOIN danh_muc dm ON sp.danh_muc_id = dm.danh_muc_id WHERE sp.trang_thai = 'con_hang' ORDER BY sp.ngay_tao DESC;

Tạo Đơn Hàng (Transaction)

BEGIN TRANSACTION; INSERT INTO don_hang (tai_khoan_id, ngay_dat, tong_tien, trang_thai, dia_chi_giao) VALUES (5, GETDATE(), 500000, 'cho_xu_ly', '123 Nguyễn Huệ'); SET @donHangId = SCOPE_IDENTITY(); -- Lấy ID vừa insert INSERT INTO chi_tiet_don_hang (don_hang_id, san_pham_id, so_luong, gia) VALUES (@donHangId, 1, 2, 150000); UPDATE san_pham SET so_luong_ton = so_luong_ton - 2 WHERE san_pham_id = 1; COMMIT TRANSACTION; -- Hoặc ROLLBACK nếu lỗi

Kiểm Tra Mã Giảm Giá

SELECT * FROM ma_giam_gia WHERE ma_code = 'SUMMER2024' AND CAST(GETDATE() AS DATE) BETWEEN ngay_bat_dau AND ngay_ket_thuc AND (so_luong = -1 OR da_su_dung < so_luong); -- so_luong = -1 có nghĩa vô hạn

Thống Kê Doanh Thu

SELECT COUNT(*) AS [Tổng Đơn], SUM(tong_tien) AS [Tổng Doanh Thu], AVG(tong_tien) AS [Đơn Trung Bình], YEAR(ngay_dat) AS [Năm] FROM don_hang WHERE trang_thai IN ('dang_giao', 'da_giao') GROUP BY YEAR(ngay_dat) ORDER BY YEAR(ngay_dat) DESC;

11. Gỡ Lỗi & Troubleshooting

🛑 Lỗi: "Cannot find module 'mssql'"

❌ Nguyên Nhân: npm install chưa hoàn tất hoặc dependencies lỗi

✅ Giải Pháp:

npm install mssql

Hoặc cài lại toàn bộ:

rm -r node_modules package-lock.json npm install

🛑 Lỗi: "Port 3000 EADDRINUSE"

❌ Nguyên Nhân: Port 3000 đã bị một process khác chiếm

✅ Giải Pháp (Windows PowerShell):

taskkill /f /im node.exe

Hoặc thay đổi port trong app.js:

const PORT = 3001; ← Thay 3000 thành 3001 app.listen(PORT, () => { console.log(`Server on http://localhost:${PORT}`); });

🛑 Lỗi: "Login failed for user"

❌ Nguyên Nhân: Sai username/password hoặc database config

✅ Giải Pháp:

  1. Kiểm tra database/schema.sql đã import vào SQL Server?
    sqlcmd -S (localdb)\MSSQLLocalDB -E
    SELECT name FROM sys.databases;
  2. Kiểm tra config/database.js có tên database đúng:
    database: 'shop_thoi_trang' ← Phải là tên đúng
  3. Nếu dùng SQL Server Auth (sa), kiểm tra password:
    authentication: { type: 'default', options: { userName: 'sa', password: 'YOUR_PASSWORD' } }

🛑 Lỗi: "Cannot connect to (localdb)\MSSQLLocalDB"

❌ Nguyên Nhân: LocalDB chưa cài, hoặc tên instance sai

✅ Giải Pháp:

  1. Kiểm tra LocalDB cài chưa:
    sqllocaldb info

    Nếu không có output → chưa cài LocalDB

  2. Cài LocalDB từ VS 2022 Installer:
    Visual Studio Installer → Modify → Data storage & processing → SQL Server Express LocalDB
  3. Hoặc cài MSSQL Developer Edition và dùng tên server khác:
    server: 'LAPTOP-MYNAME' // Tên máy // hoặc server: 'localhost'

🛑 Lỗi: "SyntaxError: Unexpected token < in JSON"

❌ Nguyên Nhân: Server trả về HTML error page thay vì JSON

✅ Giải Pháp:

  1. Kiểm tra error log từ server terminal (node app.js)
  2. Thường là database query lỗi → kiểm tra SQL syntax
  3. Thêm console.log() trong route để debug:
    router.get('/san-pham', async (req, res) => { console.log('REQUEST RECEIVED'); ← Debug point try { const pool = await getPool(); console.log('POOL OK'); const result = await pool.request().query('SELECT * FROM san_pham'); console.log('QUERY OK:', result.recordset); res.json(result.recordset); } catch (error) { console.error('ERROR:', error.message); res.status(500).json({ error: error.message }); } });

🛑 Lỗi: "SQL Server certificate validation failed"

❌ Nguyên Nhân: Chứng chỉ SSL không tin tưởng

✅ Giải Pháp: Thêm trong config/database.js:

options: { encrypt: true, trustServerCertificate: true ← Thêm dòng này }
