CHƯƠNG I: GIỚI THIỆU

1.1 Đặt vấn đề nghiên cứu

Trong bối cảnh thương mại điện tử phát triển mạnh mẽ tại Việt Nam, ngành thời trang là một trong những lĩnh vực có tốc độ tăng trưởng nhanh nhất trên các nền tảng trực tuyến. Theo thống kê, thị trường thời trang trực tuyến Việt Nam liên tục tăng trưởng với tốc độ trên 20% mỗi năm, cho thấy nhu cầu mua sắm thời trang qua Internet ngày càng lớn.

Tuy nhiên, nhiều cửa hàng thời trang vừa và nhỏ vẫn đang quản lý bằng phương pháp thủ công: ghi chép sổ sách, quản lý tồn kho bằng Excel, không có hệ thống theo dõi đơn hàng hay thống kê doanh thu tự động. Điều này dẫn đến nhiều sai sót trong quản lý, mất thời gian và khó mở rộng kinh doanh.

Bên cạnh đó, việc tích hợp các tính năng hiện đại như chatbot tư vấn thời trang thông minh, gợi ý kích thước dựa trên số đo cơ thể, hệ thống mã giảm giá... giúp nâng cao trải nghiệm khách hàng và tăng khả năng cạnh tranh.

Xuất phát từ nhu cầu thực tế đó, em đã chọn đề tài "Xây dựng Website Quản Lý Shop Thời Trang" nhằm xây dựng một hệ thống web hoàn chỉnh, đáp ứng nhu cầu quản lý của chủ cửa hàng và mua sắm của khách hàng.

1.2 Mục tiêu nghiên cứu

1.2.1 Mục tiêu chung

Xây dựng một website quản lý shop thời trang hoàn chỉnh với hai phân hệ: quản trị (admin) và người dùng (khách hàng), sử dụng công nghệ Node.js, Express.js kết hợp hệ quản trị cơ sở dữ liệu Microsoft SQL Server 2022.

1.2.2 Mục tiêu cụ thể

  • Xây dựng hệ thống xác thực và phân quyền với 3 vai trò: Admin, Nhân viên, Người dùng.
  • Xây dựng trang quản trị với đầy đủ chức năng CRUD cho: sản phẩm, danh mục, khách hàng, nhân viên, đơn hàng, phiếu nhập hàng, mã giảm giá, liên hệ.
  • Xây dựng trang Dashboard tổng quan với biểu đồ doanh thu, thống kê thời gian thực.
  • Xây dựng module thống kê và báo cáo: doanh thu theo tháng, sản phẩm bán chạy, tồn kho, khách hàng thân thiết.
  • Xây dựng trang người dùng: trang chủ, cửa hàng, giỏ hàng, đặt hàng, theo dõi đơn hàng, hồ sơ cá nhân.
  • Tích hợp AI Chatbot tư vấn thời trang: gợi ý size, xu hướng thời trang, tư vấn theo dáng người.
  • Xây dựng hệ thống tìm kiếm toàn hệ thống (9 bảng dữ liệu).

1.3 Phạm vi nghiên cứu

1.3.1 Không gian

Hệ thống được xây dựng và triển khai trên môi trường localhost, phục vụ cho mô hình cửa hàng thời trang vừa và nhỏ tại Việt Nam.

1.3.2 Phạm vi thời gian

Đồ án được thực hiện trong khuôn khổ học kỳ Đồ án Cơ sở, năm học 2025 – 2026.

1.4 Đối tượng nghiên cứu

  • Quy trình quản lý cửa hàng thời trang: sản phẩm, đơn hàng, kho hàng, khách hàng, nhân viên.
  • Công nghệ phát triển web server-side: Node.js, Express.js, EJS, SQL Server.
  • Chatbot tư vấn thời trang dựa trên quy tắc (Rule-based AI).
  • Giao diện người dùng responsive với Bootstrap 5.

CHƯƠNG II: CƠ SỞ LÝ THUYẾT

2.1 Tổng quan về Node.js và Express.js

2.1.1 Node.js

Node.js là một nền tảng phát triển ứng dụng phía server được xây dựng trên engine V8 của Google Chrome. Node.js sử dụng mô hình I/O không đồng bộ (non-blocking I/O), cho phép xử lý nhiều kết nối đồng thời với hiệu suất cao.

Ưu điểm:

  • Hiệu suất cao nhờ engine V8 và mô hình event-driven.
  • Dùng JavaScript cả frontend và backend, giúp đồng nhất ngôn ngữ lập trình.
  • Hệ sinh thái npm đồ sộ với hơn 2 triệu package.
  • Phù hợp cho ứng dụng web thời gian thực.

2.1.2 Express.js

Express.js là framework web phổ biến nhất cho Node.js, cung cấp cơ chế routing, middleware, template engine linh hoạt. Express.js giúp xây dựng ứng dụng web và API một cách nhanh chóng, có cấu trúc rõ ràng.

Trong đồ án, Express.js phiên bản 4.18.2 được sử dụng làm framework chính cho toàn bộ hệ thống.

2.2 Hệ quản trị CSDL Microsoft SQL Server 2022

Microsoft SQL Server 2022 là hệ quản trị cơ sở dữ liệu quan hệ (RDBMS) mạnh mẽ của Microsoft. SQL Server hỗ trợ ngôn ngữ truy vấn T-SQL, stored procedure, transaction, trigger và nhiều tính năng bảo mật nâng cao.

Ưu điểm:

  • Hiệu suất truy vấn cao với Query Optimizer thông minh.
  • Hỗ trợ Transaction đảm bảo tính toàn vẹn dữ liệu (ACID).
  • Bảo mật tốt với Windows Authentication và mã hóa dữ liệu.
  • Tích hợp tốt với các ứng dụng trên nền tảng Windows.

Nhược điểm: Chi phí bản quyền cao cho bản Enterprise; chủ yếu chạy trên Windows.

Trong đồ án, kết nối SQL Server sử dụng driver mssql/msnodesqlv8 qua ODBC Driver 18, xác thực bằng Windows Authentication, database tên shop_thoi_trang với collation Vietnamese_CI_AS.

2.3 Template Engine EJS

EJS (Embedded JavaScript) là template engine cho phép nhúng mã JavaScript trực tiếp vào HTML. EJS sử dụng cú pháp đơn giản với các tag <%= %> để chèn dữ liệu, <% %> để thực thi logic, cho phép render HTML phía server một cách linh hoạt.

Kết hợp với express-ejs-layouts, hệ thống có 2 layout chính: layout quản trị (main.ejs) với sidebar menu, và layout người dùng (user.ejs) với navbar trên cùng.

2.4 Bootstrap 5 và Font Awesome

Bootstrap 5.3.2 là CSS framework responsive phổ biến nhất, cung cấp hệ thống grid, components (button, card, modal, table...) và utilities giúp xây dựng giao diện nhanh chóng, tương thích đa thiết bị.

Font Awesome 6.5.0 cung cấp bộ icon vector phong phú với hơn 2.000 icon miễn phí, giúp giao diện trực quan và chuyên nghiệp hơn.

Chart.js 4.4.0 là thư viện vẽ biểu đồ JavaScript, được sử dụng để hiển thị biểu đồ doanh thu, biểu đồ tròn danh mục trên trang Dashboard và Thống kê.

2.5 Mô hình MVC (Model – View – Controller)

Hệ thống được xây dựng theo mô hình MVC, tách biệt ba thành phần chính:

  • Model: Cơ sở dữ liệu SQL Server — lưu trữ và truy xuất dữ liệu qua các câu truy vấn parameterized.
  • View: 40 file template EJS tổ chức trong thư mục views/ — hiển thị giao diện cho người dùng.
  • Controller: 15 file route trong thư mục routes/ — xử lý logic nghiệp vụ, nhận request và trả response.

CHƯƠNG IV: CHƯƠNG TRÌNH ỨNG DỤNG

4.1 Giao diện người dùng

4.1.1 Trang chủ

Trang chủ hiển thị slider banner giới thiệu, danh sách danh mục sản phẩm (10 danh mục: Áo nam, Quần nam, Váy/Đầm, Giày dép, Phụ kiện, Áo nữ, Quần nữ, Áo khoác, Đồ thể thao, Túi xách) và 8 sản phẩm mới nhất với hình ảnh, tên, giá bán.

(Chụp screenshot trang chủ tại: http://localhost:3000/trang-chu)

4.1.2 Trang cửa hàng

Trang cửa hàng cho phép khách hàng duyệt toàn bộ 48 sản phẩm với các tính năng: lọc theo danh mục, lọc theo giới tính (Nam/Nữ/Unisex), tìm kiếm theo tên, phân trang 12 sản phẩm/trang.

(Chụp screenshot tại: http://localhost:3000/cua-hang)

4.1.3 Trang đăng nhập / Đăng ký

Giao diện đăng nhập và đăng ký được thiết kế theo phong cách hiện đại với Bootstrap 5. Mật khẩu được mã hóa bằng bcrypt (salt rounds = 10) trước khi lưu vào CSDL. Hệ thống kiểm tra trùng tên đăng nhập và yêu cầu mật khẩu tối thiểu 6 ký tự.

4.1.4 Giỏ hàng

Giỏ hàng được lưu trữ trong Session (phía server), cho phép thêm sản phẩm, cập nhật số lượng, xóa sản phẩm. Khi thêm sản phẩm đã có trong giỏ, số lượng được cộng dồn tự động. Hệ thống kiểm tra tồn kho trước khi cho phép thêm.

4.1.5 Đặt hàng & Thanh toán

Quy trình đặt hàng sử dụng SQL Transaction để đảm bảo tính toàn vẹn dữ liệu:

  1. Hiển thị giỏ hàng + form thông tin giao hàng (họ tên, SĐT, địa chỉ).
  2. Cho phép nhập mã giảm giá — hệ thống kiểm tra real-time (API) tính hợp lệ, hạn dùng, đơn tối thiểu.
  3. Chọn phương thức thanh toán: Tiền mặt / Chuyển khoản / Thẻ.
  4. Xử lý: Tạo/cập nhật khách hàng → Tạo đơn hàng → Tạo chi tiết đơn → Trừ tồn kho → Xóa giỏ hàng.

4.1.6 Đơn hàng của tôi

Người dùng có thể xem danh sách tất cả đơn hàng đã đặt (phân trang 10 đơn/trang), xem chi tiết từng đơn, và hủy đơn ở trạng thái "Chờ xác nhận" (tự động hoàn lại tồn kho).

Đơn hàng có 5 trạng thái: Chờ xác nhận → Đã xác nhận → Đang giao → Đã giao / Đã hủy.

4.1.7 Hồ sơ cá nhân

Trang thông tin cá nhân cho phép người dùng cập nhật: họ tên, email, số điện thoại, địa chỉ, ngày sinh, giới tính, upload ảnh đại diện (hỗ trợ JPEG/PNG/GIF/WebP, tối đa 5MB). Đặc biệt, người dùng có thể nhập số đo hình thể (chiều cao, cân nặng, 3 vòng) để AI Chatbot tư vấn size chính xác.

Trang cũng hiển thị thống kê cá nhân: tổng đơn hàng, đơn hoàn thành, tổng chi tiêu.

4.1.8 AI Chatbot tư vấn thời trang

Chatbot AI là tính năng nổi bật được tích hợp dưới dạng widget nổi (floating) ở góc phải dưới màn hình. Chatbot hỗ trợ 8 loại ý định (intent):

Tính năngMô tả
Gợi ý sizeTính toán từ số đo cơ thể → gợi ý size áo (XS–XXL) và size quần theo bảng size chuẩn. Sử dụng hệ thống chấm điểm (scoring) với trọng số: vòng ngực +3, vòng eo +2, cân nặng +2, chiều cao +1.
Gợi ý theo dáng ngườiTính BMI → phân loại dáng (gầy/cân đối/đầy đặn) → đưa ra 5 mẹo chọn đồ + sản phẩm phù hợp từ DB.
Xu hướng thời trangGiới thiệu 8 xu hướng thời trang 2026: Minimalist Chic, Streetwear & Y2K, Smart Casual, Athleisure, Cottagecore, Dark Academia, Denim-on-denim, Color Block.
Tìm kiếm sản phẩmTìm sản phẩm trong CSDL theo keyword + ưu tiên giới tính của user.
Thông tin shopTrả lời câu hỏi về shop, giờ mở cửa, liên hệ.

4.2 Giao diện quản trị (Admin)

4.2.1 Dashboard tổng quan

Trang Dashboard hiển thị 6 thẻ KPI (tổng sản phẩm, tổng đơn hàng, tổng khách hàng, tổng doanh thu, đơn chờ xử lý, SP hết hàng), biểu đồ doanh thu 7 ngày gần nhất (Chart.js), bảng 5 đơn hàng gần nhất, bảng 5 SP bán chạy nhất.

4.2.2 Quản lý sản phẩm

Hệ thống CRUD đầy đủ cho 48 sản phẩm thuộc 10 danh mục, phân thành 3 giới tính (16 SP Nam, 20 SP Nữ, 12 SP Unisex). Hỗ trợ upload hình ảnh qua Multer, tìm kiếm, lọc theo danh mục/giới tính, phân trang.

4.2.3 Quản lý đơn hàng

Admin có thể xem danh sách đơn hàng, tạo đơn mới, cập nhật trạng thái (5 bước), xem chi tiết đơn hàng và in hóa đơn (window.print).

4.2.4 Quản lý nhập hàng

Tạo phiếu nhập hàng từ nhà cung cấp, thêm nhiều sản phẩm cùng lúc, tự động cộng tồn kho khi tạo phiếu. Xem chi tiết và xóa phiếu nhập.

4.2.5 Thống kê & Báo cáo

Module thống kê toàn diện với khả năng lọc theo tháng/năm:

  • Biểu đồ cột doanh thu 12 tháng (Chart.js).
  • Biểu đồ tròn doanh thu theo danh mục.
  • TOP 10 sản phẩm bán chạy trong tháng.
  • TOP 10 khách hàng theo chi tiêu.
  • Báo cáo tồn kho: 20 SP tồn kho thấp nhất (đánh dấu đỏ: hết hàng, vàng: sắp hết).
  • Nút "In báo cáo" — in toàn bộ trang thống kê (window.print + CSS @media print).

4.2.6 Tìm kiếm nâng cao

Tìm kiếm toàn hệ thống qua 9 bảng dữ liệu: sản phẩm, đơn hàng, khách hàng, tài khoản, nhân viên, danh mục, mã giảm giá, liên hệ, phiếu nhập. Hỗ trợ lọc theo phạm vi (tất cả hoặc từng loại).

4.2.7 Quản lý mã giảm giá

Tạo, sửa, xóa mã giảm giá với 2 loại: phần trăm và số tiền cố định. Cài đặt: giá trị giảm, đơn tối thiểu, số lượng sử dụng, thời hạn bắt đầu/kết thúc.

4.2.8 Quản lý liên hệ

Xem danh sách tin nhắn liên hệ từ khách hàng. Admin có thể xem chi tiết và gửi phản hồi. Trạng thái tự động cập nhật: Chờ xử lý → Đã phản hồi.

4.2.9 Cài đặt hệ thống

Cấu hình: cài đặt đơn hàng và thiết lập thanh toán (upload ảnh QR chuyển khoản). Dữ liệu lưu dạng key-value trong bảng cai_dat.

CHƯƠNG V: THỬ NGHIỆM VÀ ĐÁNH GIÁ

5.1 Ưu điểm

  • Hệ thống đầy đủ chức năng với 15 module route, 40 file template, hơn 7.300 dòng code.
  • Giao diện responsive, tương thích tốt trên máy tính, máy tính bảng và điện thoại nhờ Bootstrap 5.
  • Bảo mật tốt: mật khẩu mã hóa bcrypt, parameterized queries chống SQL Injection, phân quyền 3 cấp.
  • AI Chatbot tư vấn thời trang thông minh — tính năng nâng cao giúp trải nghiệm người dùng tốt hơn.
  • Hệ thống mã giảm giá linh hoạt (phần trăm/cố định, hạn dùng, đơn tối thiểu).
  • Dashboard trực quan với biểu đồ Chart.js, thống kê doanh thu đa chiều.
  • Đặt hàng sử dụng SQL Transaction đảm bảo tính toàn vẹn dữ liệu.
  • Tìm kiếm toàn hệ thống qua 9 bảng dữ liệu.
  • Quản lý nhập/xuất kho hoàn chỉnh.

5.2 Nhược điểm

  • Chatbot hoạt động dạng rule-based, chưa tích hợp AI thực sự (GPT, LLM), nên khả năng hiểu ngữ cảnh còn hạn chế.
  • Chưa có tính năng thanh toán trực tuyến (VNPay, MoMo, ZaloPay).
  • Chưa có chức năng xuất báo cáo ra file Excel/PDF (chỉ có in qua trình duyệt).
  • Giỏ hàng lưu trong Session — mất khi đóng trình duyệt hoặc hết session.
  • Chưa có hệ thống đánh giá/bình luận sản phẩm.
  • Chưa có tính năng gửi email thông báo (xác nhận đơn hàng, quên mật khẩu).

CHƯƠNG VI: KẾT LUẬN

6.1 Kết quả đạt được

Sau quá trình nghiên cứu và phát triển, đồ án đã hoàn thành các mục tiêu đề ra:

  • Xây dựng thành công website quản lý shop thời trang hoàn chỉnh với 2 phân hệ: quản trị (12 chức năng) và người dùng (9 chức năng).
  • Thiết kế cơ sở dữ liệu gồm 12 bảng với 11 mối quan hệ khóa ngoại, đảm bảo tính chuẩn hóa.
  • Tích hợp thành công AI Chatbot tư vấn thời trang với 8 loại ý định, gợi ý size dựa trên số đo cơ thể.
  • Hệ thống thống kê và báo cáo trực quan với Chart.js.
  • Tổng cộng hơn 7.300 dòng code (15 route files, 40 view templates).

6.2 Hướng phát triển trong tương lai

  • Tích hợp AI nâng cao: Thay thế chatbot rule-based bằng Google Gemini API hoặc OpenAI GPT để hiểu ngữ cảnh tốt hơn.
  • Thanh toán trực tuyến: Tích hợp cổng thanh toán VNPay, MoMo, ZaloPay cho trải nghiệm mua sắm tiện lợi hơn.
  • Ứng dụng di động: Phát triển ứng dụng mobile (React Native) hoặc PWA cho khách hàng.
  • Xuất báo cáo: Tích hợp xuất Excel (exceljs) và PDF (pdfkit) cho module thống kê.
  • Hệ thống đánh giá: Cho phép khách hàng đánh giá và bình luận sản phẩm (star rating + review).
  • Email notification: Gửi email xác nhận đơn hàng, cập nhật trạng thái, quên mật khẩu (Nodemailer + Gmail).
  • Real-time: Tích hợp Socket.IO để thông báo đơn hàng mới cho admin theo thời gian thực.
  • Deploy cloud: Triển khai trên Azure/AWS/Railway với database Azure SQL hoặc Neon PostgreSQL.

6.3 Kết luận

Đồ án "Xây dựng Website Quản Lý Shop Thời Trang" đã hoàn thành đầy đủ các yêu cầu đặt ra, xây dựng một hệ thống web hoàn chỉnh phục vụ cho việc quản lý cửa hàng thời trang và mua sắm trực tuyến. Hệ thống sử dụng công nghệ Node.js, Express.js, EJS và SQL Server 2022 — những công nghệ hiện đại, phổ biến trong ngành phát triển web.

Đặc biệt, tính năng AI Chatbot tư vấn thời trang là điểm sáng của đồ án, thể hiện khả năng tích hợp trí tuệ nhân tạo vào ứng dụng thực tế, giúp nâng cao trải nghiệm người dùng và tạo ra giá trị khác biệt cho sản phẩm.

Mặc dù vẫn còn một số hạn chế cần khắc phục, đồ án đã đạt được mục tiêu ban đầu và có thể tiếp tục phát triển mở rộng trong tương lai.
